# 半導体工場シミュレーター (Semi Factory Simulator)

半導体工場の製造ラインを組んで回すシミュレーションゲームのプロトタイプ。
経営・マネタイズではなく「ラインを作って流す」工程そのものを楽しむタイプの
ゲームを目指しています(Factorio / Minecraft工業mod系が好きな人向け)。

白基調のクリーンルームを**3D(Three.js)**で「精密ジオラマ」調に描画します。
壁とFFU天井(発光パネル)に囲まれた部屋の中でラインを組み、AgXトーン
マッピング+IBL+GTAO(AO)+ブルーム+ティルトシフトの描画パイプラインが
ミニチュア写真のような見た目を作ります。壁は視点の手前に来ると自動的に
消えます(ドールハウス表示)。視点を自由に回転・ズームしながら、
**装置の配置**と**天井搬送(OHT)レールの敷設**でラインを設計します。
**Shiftキー**(またはツールバーのボタン)で、真上固定の**2Dモード**
(直交投影・ポスト効果なしの精密作業モード)にいつでも切り替えられます。
レールを正確に敷きたいときは2Dモードが向いています。

## コアメカニクス

### 資金

初期資金は **¥30,000**。装置の設置・OHTビークルの購入・レールの敷設
(¥25/区間)・**技術ツリーの研究**に資金がかかり、ロット投入時にはウェハ原価を
支払います(先端ノードほど原価も高い)。収入は出荷売上のみ:

> **売上 = 基準単価 × ノード倍率 × 市場鮮度 × 歩留まり**

**ノード倍率**は微細なノードほど大きく(2nmで約34倍)、**市場鮮度**は自分が
そのノードで量産するほど値崩れします(後述)。装置の撤去・OHTの売却・レールの
撤去は半額払い戻し。廃棄したロットの原価は戻らないので、歩留まりの悪化と詰まりは
そのまま損失になります。

### 技術ツリー(研究)

微細化も製品解禁も**プレイヤー主導の研究**で進みます(トップバー「研究」ボタン
または **G** キーでパネルを開く)。研究は**同時1件**。着手すると資金を支払い、
一定時間(シミュ内数十秒〜数百秒)後に解禁されます。研究中も現行ラインで稼げるので、
「稼ぎながら次の技術を仕込む」時間の重なりが生まれます。中止すると費用の半額が戻ります。

- **プロセスノード**は 250→180→…→2nm の**一本のチェーン**。前のノードを研究済みだと
  次を研究できます。研究済みノードは**全製品で生産に使えます**。
- **製品**は前提条件つき(例: DRAM = ロジックIC研究済み + 130nmノード解禁、
  GPU = CPU研究済み + 7nmノード解禁)。

### プロセスノードと生産(どの世代で作るか)

プロセスノードは **14世代**(250 / 180 / 130 / 90 / 65 / 45 / 32 / 22 / 14 / 10 / 7 /
5 / 3 / 2nm)。研究済みノードの中から、**投入ステーションごとに生産ノードを選べます**。

ノードが進むほど:
- **配線層が増える**(約3世代ごとに +1層。ダマシン5工程 = 成膜→露光→エッチ→
  メタル→CMP が検査直前に追加)
- 節目で**FEOL工程が加わる**(45nm: ゲート絶縁膜ALD / 22nm: FinFET / 3nm: GAA)
- 微細な露光は**マルチパターニング**(露光を複数回)や**上位露光機**を要求する

工程が伸びるほど欠陥が積み増され、歩留まり維持とライン能力の両方が問われます。
古いノードは工程が短く歩留まりも安定するぶん単価が安く、先端ノードは重いが高単価
——「枯れたノードで薄利多売」対「先端で高マージン少量」が本物のトレードオフになります。

### 市場鮮度(値崩れ)と歩留まり学習

進行が「せわしなく」ならないよう、2本のカーブでノードの旨味の窓を作ります:

- **市場鮮度(下がる)**: 先端ノードは希少で高価格。**そのノード×製品で自分が出荷した
  数**が増えるほどなめらかに値崩れし、製品ごとの床値で下げ止まります
  (段差ではなく指数減衰なので、せわしない最適化を招きません)。
- **歩留まり学習(上がる)**: 新ノードは立ち上げ直後ほど欠陥が多く、そのノードでの
  累計生産が増えるほど改善します(実工場のyield ramp。ノード単位でファブ共通なので、
  安い製品で習熟を上げてから旗艦製品を投入する戦略が効きます)。

工程フローパネル(F)に、選択中の製品×ノードの**市場鮮度**と**1ロットあたり粗利**が
表示されます。投入ステーションの設定でも粗利見込みを確認できます。

### FEOLとBEOL

工程は実工場と同じ2部構成です:

- **FEOL(前半: トランジスタ形成)** — 酸化(炉)・露光・エッチング・**イオン注入**・
  アニール(炉)。45nm以降は**ゲート絶縁膜ALD**、22nm以降はFinFET工程が加わります
- **BEOL(後半: 配線形成)** — CVD成膜・露光・エッチング・**メタル成膜**・**CMP平坦化**。
  配線1層ごとにこの5工程(ダマシン)を繰り返します

露光がFEOL/BEOL両方から再訪される第1のハブ、CMPが配線層の数だけ再訪される第2の
ハブです。注入=上流専用/メタル・CMP=下流専用という性質があるので、FEOLベイと
BEOLベイに分けて配置すると搬送が整理されます。CMPはスラリーで最も汚れやすい装置です。

### 装置ティア(露光・メタル)

現実の技術進化に沿って、いくつかの工程は世代準拠のティアを持ちます。同じ工程を担う
装置は搬送も混雑分散も自動で合流します。

**露光(5ティア)** — 各ティアには「単パターンの限界」と「MPを駆使した絶対限界」があり、
どこかで必ず上位機が必要になります:

| 装置 | 価格 | 単パターン限界 | 絶対限界(MP) |
|---|---|---|---|
| i線ステッパ | ¥8,000 | 180nm | 130nm |
| KrFスキャナ | ¥20,000 | 90nm | 65nm |
| ArF液浸スキャナ | ¥60,000 | 32nm | 7nm |
| EUV露光装置 | ¥180,000 | 5nm | 2nm |
| High-NA EUV | ¥400,000 | 2nm | — |

強制交代点は **90nm(KrF必須)→ 45nm(ArF液浸必須)→ 5nm(EUV必須)**。2nmは
「EUVをMP(露光2回)で凌ぐか、High-NAで1回で写すか」の経済判断です。投入ステーションで
露光技術を選ぶと、MP世代ではレシピの露光工程が2〜4倍に増えます(安い装置で工程を稼ぐか、
高い装置で1回で写すか)。

**メタル(2ティア)** — Alスパッタ(PVD、〜65nm)と**Cuめっき(ECD、130nm〜全世代)**。
45nm以降の微細配線はAlでは不可で、Cuめっき装置が必須になります。

### 拡散炉(バッチ処理)

酸化・アニール工程を担う**バッチ炉**。最大3ロットを同時処理でき、満載で焼くほど
効率的ですが、満載を待ちすぎるとフローが淀みます(装填後10秒で自動的に処理開始)。
処理中でも次バッチの装填(プリステージ)は進みます。

### 装置とロードポート

装置には実物準拠のフットプリントがあります(露光・イオン注入・炉の3×2が最大、
検査2×1が最小)。各装置の南面には **ロードポート**(IN/OUT)があり、FOUPはそこで
搬送待ちの間ストックされます。処理完了品は出力ポートが空くまで装置内に留まり、
その間は次の処理を始められません(積層灯が赤になります)。

### 投入・出荷ステーション(複数ライン)

投入・出荷ステーションは**複数設置**できます。**投入ステーション1台=1本の生産ライン
設定**(製品・生産ノード・露光技術・投入間隔)で、装置を選んでカードから設定します。
これで「ダイオード専用の枯れライン」と「EUVの先端ライン」を並走させるなど、製品の
作り分けがそのままフロア上のラインとして表現されます。出荷ステーションは最寄り
(レール経路が最短)へ搬送されます。

### OHT(天井搬送)

- レールはドラッグで敷設。**敷いた方向にしか走れない一方通行**です
- ロードポートの真上にレールが通っていないと搬送できません(⚠警告が出ます)
- ビークルは搬送ジョブに応じて自動で稼働しますが、**保有台数まで**しか湧きません。
  足りなければトップバーの[+]で買い足します
- 一方通行なので、単一の大きなループだけだと毎回ほぼ1周する羽目になります。
  **南北の連絡線(ショートカット)を足して小さなループの網にする**のが基本です
- ビークルは**渋滞回避ルーティング**で走ります(待たされた区間は経路コストが上がり、
  迂回路があればそちらを選ぶ)

### ストッカーとデッドロック対策

リエントラントフローでは循環待ち(デッドロック)が本当に起きます。実工場と同じ仕組みで
対処しています:

1. **ストッカー退避**: 行き先が満杯のFOUPは自動でストッカー(6枠)へ退避
2. **WIPリリース制御(CONWIP)**: ストッカーが全て満杯の間は新規投入を停止
3. **ディスパッチ優先則**: 工程が進んだロットから搬送(下流から抜く)
4. **アイドルOHTのポート回避**: 空車のOHTは装置のロードポート上に居座らない

完全に凍結した場合の脱出弁として、装置を選択して**「滞留ロット廃棄」**を押すと、
その装置に滞留しているFOUPを強制廃棄して循環待ちを断ち切れます。

出力ポートのFOUPが次工程へ運べず滞留すると、装置カードと3Dの「!」警告で理由を
知らせます:**レール経路がない** / **次工程の装置が未設置**(露光ティア不足・Cu未設置・
ALD未設置を含む)。単に行き先が満杯で空き待ちなだけの正常な滞留には警告を出しません。

### 歩留まりと自動メンテナンス(PM)

各工程で少しずつ欠陥が乗り、装置は使うほど汚れて欠陥率が上がります(汚れMAXで
欠陥率は約4.2倍)。**故障とランダム停止・手動修理は廃止**しました。代わりに清浄度が
閾値(50%)を下回ると、ジョブ完了後に装置が**自動で予防保全(PM)**に入り
(15秒停止・清浄度全回復)、クリック不要でリフレッシュします。CMPのように汚れやすい
装置は自然にPM頻度が高く=実効能力が低い、という個性が数字として残ります。汚れ対策の
基本は同種装置の複数台運用による負荷分散です。検査時点で歩留まり40%未満のロットは
廃棄されます。

### 統計とボトルネック診断

- トップバー: 資金/スループット/歩留まり(直近30ロットの移動平均)/平均搬送待ち/整備中数
- 工程フローパネル(F): 製品×ノードの工程を表示。**繰り返し部分は「×N」で省略表示**
  (配線層やマルチパターニングが畳まれる)。工程ごとに装置稼働率(85%以上は赤=
  ボトルネック候補)と仕掛かり数、市場鮮度と粗利を表示。**工程をクリックするとフロア上の
  該当装置がリングでハイライト**されます
- 技術ツリーパネル(G): 研究の進行と解禁状況
- 渋滞ヒートマップ(H): ビークルが待たされた場所ほど赤く光る

### セーブ

10秒ごとにlocalStorageへ自動保存され、次回起動時に復元されます。トップバーの「データ」
からJSONの書き出し/読み込み、新規工場の開始が可能。旧バージョンのセーブも読み込めますが、
システム刷新(技術ツリー化・露光ティア拡張)をまたぐ場合、仕掛かり中のロットだけは
リセットされます(工場・資金・研究進捗は維持。到達済み世代までのノードは研究済みへ、
旧露光機DUVはKrFへ引き継がれます)。

## 操作

| 操作 | 効果 |
|---|---|
| 建設メニュー(下部): カテゴリタブ + 装置 | ツール切替。固定ツール 1:選択 / 2:レール / 3:レール撤去 / X:装置撤去 |
| 右クリック(設置ツール中) / R | 設置する装置を時計回りに回転(ポート面: 南→西→北→東) |
| 左ドラッグ(レールツール) | レール敷設(ドラッグ方向=進行方向) |
| 左クリック(選択ツール) | 装置を選択(状態確認 / 生産ライン設定 / 滞留ロット廃棄 / 撤去) |
| 右ドラッグ(3Dモードのみ) | 視点回転(オービット) |
| 中ドラッグ | 画面パン(矢印キーでも可) |
| ホイール | ズーム |
| **Shift** | 2D(直交・真上固定)⇔3D 表示切替 |
| F | 工程フローパネル表示切替 |
| G | 技術ツリー(研究)パネル表示切替 |
| H | 渋滞ヒートマップ表示切替 |
| Esc | ツール解除 |

トップバー: 既定投入間隔スライダー / 一時停止 / 1x・2x・4x速度 / 研究 / OHT買い足し

## 開発

```bash
npm install
npm run dev    # 開発サーバー
npm run build  # 型チェック + ビルド
```

- `src/config.ts` — 装置定義・レシピ生成・ノードテーブル・技術ツリー・露光ティア・チューニング定数
- `src/sim.ts` — 工場シミュレーション(装置・ポート・ディスパッチ・研究・鮮度/学習・PM・シリアライズ)
- `src/rail.ts` — レール有向グラフと経路探索
- `src/oht.ts` — OHTビークルの運行(走行・ホイスト・渋滞ヒート)
- `src/three/scene.ts` — 3Dシーン基盤(レンダラー・軌道カメラ・照明・床・ピッキング・2D/3Dカメラ切替)
- `src/three/view3d.ts` — ゲーム状態→3Dシーンの同期(装置・レール・ビークル・オーバーレイ)
- `src/view.ts` — 描画とUIが共有するビュー状態
- `src/ui.ts` — HUD(建設メニュー・技術ツリー・ライン設定・工程フロー・コンテキストカード)
- `src/save.ts` — セーブ/ロード(localStorage・JSONファイル)
- `src/main.ts` — 入力(Pointer Events)・ゲームループ・自動保存

描画品質はURLパラメータで選べます: `?fx=high`(既定)/ `?fx=med`(ブルームのみ)/
`?fx=low`(ポストプロセス・影・AAなし)。重いと感じたら下げてください。
