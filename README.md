# 半導体工場シミュレーター (Semi Factory Simulator)

半導体工場の製造ラインを組んで回すシミュレーションゲームのプロトタイプ。
経営・マネタイズではなく「ラインを作って流す」工程そのものを楽しむタイプの
ゲームを目指しています(Factorio / Minecraft工業mod系が好きな人向け)。

白基調のクリーンルームを真上見下ろし(+高さ表現)で描き、プレイヤーは
**装置の配置**と**天井搬送(OHT)レールの敷設**でラインを設計します。

## コアメカニクス

### 工程レシピ(リエントラントフロー)

ウェハ25枚入りの **FOUP**(搬送ポッド)単位でロットが流れます。
半導体製造の最大の特徴は、同じ種類の装置を何度も再訪する
**リエントラントフロー**。レシピは9工程で成膜・露光・エッチングを2周します:

```
初期洗浄 → 成膜① → 露光① → エッチング① → 成膜② → 露光② → エッチング② → 最終洗浄 → 検査
```

### 装置とロードポート

装置には実物準拠のフットプリントがあります(露光3×2が最大、検査2×1が最小)。
各装置の南面には **ロードポート**(IN/OUT)があり、FOUPはそこで搬送待ちの間
ストックされます。処理完了品は出力ポートが空くまで装置内に留まり、
その間は次の処理を始められません(積層灯が赤になります)。

### OHT(天井搬送)

- レールはドラッグで敷設。**敷いた方向にしか走れない一方通行**です
- ロードポートの真上にレールが通っていないと搬送できません(⚠警告が出ます)
- ビークルは搬送ジョブに応じて自動で稼働しますが、**保有台数まで**しか
  湧きません。足りなければトップバーの[+]で買い足します
- 一方通行なので、単一の大きなループだけだと毎回ほぼ1周する羽目になり
  搬送が極端に遅くなります。**南北の連絡線(ショートカット)を足して
  小さなループの網にする**のがライン設計の基本です

### ストッカーとデッドロック

リエントラントフローでは「成膜→露光→エッチングが互いの空きを待つ」
循環待ち(デッドロック)が本当に起きます(実工場でも既知の問題です)。
行き先が満杯のFOUPは自動で **ストッカー**(内部保管6枠)へ退避されるので、
ラインの中間にストッカーを置いておくのが安定稼働のコツです。

### 歩留まりとメンテナンス

各工程で少しずつ欠陥が乗り、装置は使うほど汚れて欠陥率が上がります。
装置を選択して「メンテナンス」(8秒停止)で回復。検査時点で歩留まり40%未満の
ロットは廃棄されます。

### 故障

汚れた装置ほど故障しやすくなります(ジョブ完了ごとに判定)。故障した装置は
積層灯が赤点滅して停止し、選択して「修理」(25秒)するまで復帰しません。
修理後は清浄度も全回復(オーバーホール扱い)。「こまめな予防メンテで故障を
避ける」か「限界まで回して長い修理を食らう」かのトレードオフです。

### 統計と渋滞ヒートマップ

- トップバー: スループット/歩留まり/平均搬送待ち/故障数
- 工程フローパネル(F): 工程別仕掛かり+スループット推移スパークライン
- 渋滞ヒートマップ(H): ビークルが待たされた場所ほど赤く光る。
  レール増強の判断材料

### セーブ

10秒ごとにlocalStorageへ自動保存され、次回起動時に復元されます。
トップバーの「データ」からJSONの書き出し/読み込み、新規工場の開始が可能。

## 操作

| 操作 | 効果 |
|---|---|
| ホットバー(キー1〜9, 0) | ツール切替(選択/レール/レール撤去/装置5種/ストッカー/装置撤去) |
| R | 設置する装置の回転(ポート面: 南→西→北→東) |
| 左ドラッグ(レールツール) | レール敷設(ドラッグ方向=進行方向) |
| 左クリック(選択ツール) | 装置を選択(メンテ/修理/撤去/状態確認) |
| 右ドラッグ / 中ドラッグ / Space+ドラッグ | 画面パン |
| ホイール | ズーム |
| F | 工程フローパネル表示切替 |
| H | 渋滞ヒートマップ表示切替 |
| Esc | ツール解除 |

トップバー: 投入間隔スライダー / 一時停止 / 1x・2x・4x速度 / OHT買い足し

## 開発

```bash
npm install
npm run dev    # 開発サーバー
npm run build  # 型チェック + ビルド
```

- `src/config.ts` — 装置定義・レシピ・回転ヘルパー・チューニング定数
- `src/sim.ts` — 工場シミュレーション(装置・ポート・ディスパッチ・歩留まり・故障・シリアライズ)
- `src/rail.ts` — レール有向グラフと経路探索
- `src/oht.ts` — OHTビークルの運行(走行・ホイスト・渋滞ヒート)
- `src/render.ts` — Canvas描画(クリーンルーム・2.5D・ヒートマップ)
- `src/ui.ts` — HUD(ホットバー・ステータス・コンテキストカード・スパークライン)
- `src/save.ts` — セーブ/ロード(localStorage・JSONファイル)
- `src/main.ts` — 入力・カメラ・ゲームループ・自動保存

## ロードマップ

- 製品の複数種類化とテックツリー(ダイオード → ロジック → DRAM → CPU)、混流生産
- バッチ炉(酸化/アニール: 複数ロット同時処理)
- STB(レール脇バッファ)、空車の待避線・駐機スポット
- リワークループ(露光後検査 → 再露光)
- ベイ/チェイスのゾーニングとクリーン度の空間シミュレーション
- 工程レシピをプレイヤーが設計できるレシピエディタ
