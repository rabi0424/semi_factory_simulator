# 半導体工場シミュレーター (Semi Factory Simulator)

半導体工場の製造ラインを組んで回すシミュレーションゲームのプロトタイプ。
経営・マネタイズではなく「ラインを作って流す」工程そのものを楽しむタイプの
ゲームを目指しています(Factorio / Minecraft工業mod系が好きな人向け)。

白基調のクリーンルームを**3D(Three.js)**で描画し、視点を自由に回転・
ズームしながら、**装置の配置**と**天井搬送(OHT)レールの敷設**でライン
を設計します。天井を走るOHTがワイヤーホイストでFOUPを吊り上げ/吊り下ろす
様子が立体で見えます。**Shiftキー**(またはツールバーのボタン)で、真上
固定の**2Dモード**(直交投影)にいつでも切り替えられます。3Dはパースの
せいで装置の真上とレールの位置がズレて見えることがあるため、レールを
正確に敷きたいときは2Dモードが向いています。

## コアメカニクス

### 製品テックツリーと混流生産

ウェハ25枚入りの **FOUP**(搬送ポッド)単位でロットが流れます。
半導体製造の最大の特徴は、同じ種類の装置を何度も再訪する
**リエントラントフロー**。製品は4種類あり、累計完成ロット数で解禁されます:

| 製品 | 工程数 | 解禁 | 特徴 |
|---|---|---|---|
| ダイオード | 5 | 最初から | 炉1回・露光1回の入門レシピ |
| ロジックIC | 9 | 累計6 | 成膜/露光/エッチを2周(リエントラント) |
| DRAM | 13 | 累計30 | 露光3周+炉2回 |
| CPU | 17 | 累計70 | 露光4周+炉2回。歩留まり維持が困難 |

工程数は基本レシピ(250nm世代)の値。量産で微細化すると各製品の工程数は増えます
(後述「プロセスノード微細化」)。

投入ステーションを選択すると製品ごとの**投入比率**を設定でき、
複数製品の**混流生産**ができます(FOUPの下部バンドが製品識別色)。

### プロセスノード微細化

同じ製品でも量産を重ねると**プロセスノードが微細化**します(250nm→180nm→
…→45nm)。各製品を累計20ロット完成させるごとに1世代進み、**配線層が1層増える**
(成膜→露光→エッチングの1サイクルが最終検査の直前に追加される)。工程が伸びる
ほど欠陥が積み増され、歩留まりの維持とライン能力の両方が問われます。工程フロー
パネル(F)の製品タブに現在のノードが表示され、微細化の瞬間はメッセージで通知
されます。既に流れているロットは投入時の世代のレシピのまま完了します。

### 拡散炉(バッチ処理)

酸化・アニール工程を担う**バッチ炉**。最大3ロットを同時処理でき、
満載で焼くほど効率的ですが、満載を待ちすぎるとフローが淀みます
(装填後10秒で自動的に処理開始)。

### 装置とロードポート

装置には実物準拠のフットプリントがあります(露光3×2が最大、検査2×1が最小)。
各装置の南面には **ロードポート**(IN/OUT)があり、FOUPはそこで搬送待ちの間
ストックされます。処理完了品は出力ポートが空くまで装置内に留まり、
その間は次の処理を始められません(積層灯が赤になります)。

### OHT(天井搬送)

- レールはドラッグで敷設。**敷いた方向にしか走れない一方通行**です
- ロードポートの真上にレールが通っていないと搬送できません(⚠警告が出ます)
- ビークルは搬送ジョブに応じて自動で稼働しますが、**保有台数まで**しか
  湧きません。足りなければトップバーの[+]で買い足します
- 一方通行なので、単一の大きなループだけだと毎回ほぼ1周する羽目になり
  搬送が極端に遅くなります。**南北の連絡線(ショートカット)を足して
  小さなループの網にする**のがライン設計の基本です

### ストッカーとデッドロック対策

リエントラントフローでは「成膜→露光→エッチングが互いの空きを待つ」
循環待ち(デッドロック)が本当に起きます(実工場でも既知の問題です)。
このゲームでは実工場と同じ3つの仕組みで対処しています:

1. **ストッカー退避**: 行き先が満杯のFOUPは自動でストッカー(6枠)へ退避
2. **WIPリリース制御(CONWIP)**: ストッカーが全て満杯の間は新規投入を停止
3. **ディスパッチ優先則**: 工程が進んだロットから搬送(下流から抜く)
4. **アイドルOHTのポート回避**: 空車のOHTは装置のロードポート上に居座らない
   (居座ると隣のOHTと出口を塞ぎ合い、その装置が永久に積み下ろし不能になる
   ため)。空いている非ポートの行き先を優先して離脱する

それでも詰まったら、ストッカーの増設・装置の増強・投入間隔の調整で
ラインを立て直せます。

出力ポートのFOUPが次工程へ運べず滞留すると、装置カードと3Dの「!」警告で
理由を知らせます:**レール経路がない**/**次工程の装置が未設置**/**同種装置が
全台停止中(故障・整備)**。単に行き先が満杯で空き待ちなだけの正常な滞留には
警告を出しません。

### 歩留まりとメンテナンス

各工程で少しずつ欠陥が乗り、装置は使うほど汚れて欠陥率が上がります。
装置を選択して「メンテナンス」(8秒停止)で回復。検査時点で歩留まり40%未満の
ロットは廃棄されます。

### 故障

汚れた装置ほど故障しやすくなります(ジョブ完了ごとに判定)。故障した装置は
積層灯が赤点滅して停止し、選択して「修理」(25秒)するまで復帰しません。
修理後は清浄度も全回復(オーバーホール扱い)。「こまめな予防メンテで故障を
避ける」か「限界まで回して長い修理を食らう」かのトレードオフです。

### 統計と渋滞ヒートマップ

- トップバー: スループット/歩留まり/平均搬送待ち/故障数
- 工程フローパネル(F): 工程別仕掛かり+スループット推移スパークライン
- 渋滞ヒートマップ(H): ビークルが待たされた場所ほど赤く光る。
  レール増強の判断材料

### セーブ

10秒ごとにlocalStorageへ自動保存され、次回起動時に復元されます。
トップバーの「データ」からJSONの書き出し/読み込み、新規工場の開始が可能。

## 操作

| 操作 | 効果 |
|---|---|
| ホットバー(1〜3: ツール, 4〜9: 装置, 0: ストッカー, X: 撤去) | ツール切替 |
| 右クリック(設置ツール中) / R | 設置する装置を時計回りに回転(ポート面: 南→西→北→東)。ゴーストのIN(グレー)/OUT(紫)の点で向きを確認できる |
| 左ドラッグ(レールツール) | レール敷設(ドラッグ方向=進行方向) |
| 左クリック(選択ツール) | 装置を選択(メンテ/修理/撤去/状態確認) |
| 右ドラッグ(3Dモードのみ) | 視点回転(オービット) |
| 中ドラッグ | 画面パン(矢印キーでも可) |
| ホイール | ズーム |
| **Shift** | 2D(直交・真上固定)⇔3D 表示切替。トップバーの「3D表示/2D表示」ボタンでも可 |
| F | 工程フローパネル表示切替 |
| H | 渋滞ヒートマップ表示切替 |
| Esc | ツール解除 |

装置を選択し直す(ホットバーで別の種類を選ぶ、または同じ種類を選び直す)と
回転は南向き(手前=IN/OUT)にリセットされます。

トップバー: 投入間隔スライダー / 一時停止 / 1x・2x・4x速度 / OHT買い足し

## 開発

```bash
npm install
npm run dev    # 開発サーバー
npm run build  # 型チェック + ビルド
```

- `src/config.ts` — 装置定義・レシピ・回転ヘルパー・チューニング定数
- `src/sim.ts` — 工場シミュレーション(装置・ポート・ディスパッチ・歩留まり・故障・シリアライズ)
- `src/rail.ts` — レール有向グラフと経路探索
- `src/oht.ts` — OHTビークルの運行(走行・ホイスト・渋滞ヒート)
- `src/three/scene.ts` — 3Dシーン基盤(レンダラー・軌道カメラ・照明・床・ピッキング・2D/3Dカメラ切替)
- `src/three/view3d.ts` — ゲーム状態→3Dシーンの同期(装置・レール・ビークル・オーバーレイ)
- `src/view.ts` — 描画とUIが共有するビュー状態
- `src/ui.ts` — HUD(ホットバー・ステータス・コンテキストカード・スパークライン)
- `src/save.ts` — セーブ/ロード(localStorage・JSONファイル)
- `src/main.ts` — 入力(Pointer Events)・ゲームループ・自動保存

`?lowfx` をURLに付けると影とアンチエイリアスを切った軽量モードで起動します。

## ロードマップ

- 3Dルック開発(クリーンルームの壁・FFU天井・材質・モーションイージング・環境音)
- タッチ操作(1本指ツール/2本指カメラ)とレスポンシブHUD
- STB(レール脇バッファ)、空車の待避線・駐機スポット
- リワークループ(露光後検査 → 再露光)
- ベイ/チェイスのゾーニングとクリーン度の空間シミュレーション
- 工程レシピをプレイヤーが設計できるレシピエディタ
